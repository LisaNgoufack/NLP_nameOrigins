== Présentation générale du projet

=== Contexte et objectifs

Le projet *NLP-nameOrigins* s'inscrit dans le cadre d'un travail pratique de NLP. L'objectif central est d'exploiter une base de données de noms de famille français avec leurs textes étymologiques pour :

* Identifier automatiquement les variantes orthographiques et phonétiques d'un même patronyme
* Regrouper ces variantes en clusters cohérents
* Enrichir la base avec des données de fréquence INSEE
* Étendre la base aux prénoms via scraping web
* Offrir une interface de recherche conviviale

=== Problématique

[quote]
____
Comment identifier automatiquement les variantes d'un nom de famille français et enrichir une base étymologique incomplète à partir de sources hétérogènes ?
____

La difficulté centrale est qu'un même patronyme peut s'écrire de multiples façons selon les régions, les époques et les migrations :

[cols="1,2,2", options="header"]
|===
| Type de variation | Exemples | Cause

| Phonétique
| Durrand → Durand
| Erreur orthographique historique

| Régionale
| Lefebvre / Lefeuvre / Lefèvre
| Dialectes régionaux

| Composés
| Le Goff / Legoff / Le-Goff
| Agglutination ou séparation

| Latinisation
| Durandus → Durand
| Registres ecclésiastiques

| Migration
| Garcia / Garcias
| Adaptation phonétique
|===

=== Architecture modulaire

Le projet a été refactorisé selon une *architecture orientée objet modulaire* pour améliorer la maintenabilité, la testabilité et les performances.

==== Principes de conception

* *Point d'entrée unique* : `main.py` orchestre l'exécution de tout le pipeline
* *Configuration centralisée* : `src/config.py` gère toutes les variables d'environnement et paramètres
* *Services indépendants* : Chaque étape du pipeline = 1 classe dans `src/services/`
* *Caching intelligent* : Skip automatique si le fichier de sortie existe déjà (sauf `--force`)
* *Prompts externalisés* : Fichiers `.txt` modifiables sans toucher au code Python
* *Utilitaires réutilisables* : Fonctions communes dans `src/utils/`

[source,python]

class NLPNameOriginsPipeline:
    def __init__(self, config: Config):
        self.config = config
        # Initialisation des services
    
    def run(self, steps_to_run: List[int], force: bool = False):
        # Exécute les étapes demandées dans l'ordre
        for step in steps_to_run:
            self._run_step(step, force)

== Étape 1 — Clustering par Union-Find

=== Objectif

Regrouper automatiquement les noms de famille qui partagent le même texte d'origine. Si `durand` et `durant` partagent l'identifiant `O1234`, ils sont variantes l'un de l'autre.

=== Algorithme Union-Find

L'algorithme Union-Find (ou *Disjoint Set Union*) est une structure de données qui maintient une partition d'un ensemble en groupes disjoints. Il supporte deux opérations en quasi-temps constant :

* *Find(x)* : trouver le représentant du groupe de x
* *Union(x, y)* : fusionner les groupes de x et y

[source,python]
----
class UnionFind:
    def __init__(self, elements):
        self.parent = {e: e for e in elements}
        self.rank   = {e: 0 for e in elements}

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # compression de chemin
        return self.parent[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
----

=== Application au projet

[source,python]
----
# Construire l'index : origin_id → liste de noms
origin_to_names = defaultdict(list)
for entry in names:
    for oid in entry["origins"]:
        origin_to_names[oid].append(entry["name"])

# Appliquer Union-Find
uf = UnionFind(all_names)
for oid, noms in origin_to_names.items():
    for i in range(1, len(noms)):
        uf.union(noms[0], noms[i])

# Construire les clusters
clusters = defaultdict(list)
for nom in all_names:
    clusters[uf.find(nom)].append(nom)
----

import csv

noms_insee = {}
with open("data/noms2008nat_txt.txt", encoding="latin-1") as f:
    reader = csv.DictReader(f, delimiter="\t")
    for row in reader:
        nom = row["NOM"].strip().lower()

        # Filtrer la ligne "Autres noms"
        if nom == "autres noms":

        frequence = sum(
            int(v) for k, v in row.items()

            k: int(v) for k, v in row.items()
            if k != "NOM" and v.isdigit()

        noms_insee[nom] = {
            "frequence": frequence,
            "evolution": evolution

== Étape 5 — Correspondances phonétiques et orthographiques

=== Objectif

Pour les 201 771 noms INSEE sans correspondance Geneanet directe, trouver un nom Geneanet proche par la phonétique (Soundex) ou l'orthographe (Levenshtein).

=== Algorithme Soundex

L'algorithme *Soundex* encode un mot en un code phonétique de 4 caractères : la première lettre suivie de 3 chiffres représentant les consonnes.

[source,python]
----
import jellyfish

jellyfish.soundex("DURAND")   # → "D653"
jellyfish.soundex("DURANT")   # → "D653"  (même code !)
jellyfish.soundex("DURAN")    # → "D650"
jellyfish.soundex("MARTIN")   # → "M635"
jellyfish.soundex("MARTINEZ") # → "M635"  (même code !)
----

*Règles de codage Soundex :*

[cols="1,3", options="header"]
|===
| Chiffre | Consonnes représentées

| 1 | B, F, P, V
| 2 | C, G, J, K, Q, S, X, Z
| 3 | D, T
| 4 | L
| 5 | M, N
| 6 | R
|===

=== Distance de Levenshtein

La *distance de Levenshtein* mesure le nombre minimal d'opérations (insertion, suppression, substitution) pour transformer une chaîne en une autre.

[source,python]
----
from Levenshtein import distance

distance("durand",  "durrand")  # → 1  (suppression d'un 'r')
distance("lefebre", "lefebvre") # → 1  (insertion d'un 'v')
distance("martin",  "martins")  # → 1  (suppression du 's')
distance("dupont",  "dupond")   # → 1  (substitution t→d)
----

=== Pipeline de traitement

[source,python]
----
for nom_insee in noms_sans_origine:
    # 1. Essai Soundex
    code = jellyfish.soundex(nom_insee.upper())
    candidats = soundex_index.get(code, [])
    if candidats:
        meilleur = min(candidats, key=lambda c: distance(nom_insee, c))
        if distance(nom_insee, meilleur) <= 3:
            assigner_origine(nom_insee, meilleur, source="soundex")
            continue

    # 2. Essai Levenshtein étendu (distance ≤ 2)
    for rep_geneanet in tous_les_representants:

    # 3. Génération par Mistral (si toujours sans origine)

== Étape 6 — Extraction NER

=== Objectif

Les textes étymologiques mentionnent souvent des variantes et des noms associés qui ne figurent pas encore dans la base. L'objectif est de les extraire automatiquement.

Exemple de texte source :

[source,text]
----
"Variante de Bidault (voir ce nom) assez fréquente en Bretagne (29, 56).
Formes voisines : Bideaud (25, 38, 44), Bideault, Bideaut (71), Bideaux (02, 76, 90)."
----

Noms à extraire : `Bidault`, `Bideaud`, `Bideault`, `Bideaut`, `Bideaux`

=== Méthode 1 — Regex + majuscules

==== Principe

Détecter les tokens en majuscules précédés de mots-clés indicateurs de variantes :

[source,python]
----
MOTS_CLES = [
    "Variante de", "Voir aussi", "Voir ce nom",
    "Formes voisines", "Forme voisine",
    "Également écrit", "On trouve aussi",
    "Autres formes", "Forme bretonne",
]

PATTERN_NOM = r'\b[A-ZÀ-Ü][a-zà-ü]{2,}(?:\s[A-ZÀ-Ü][a-zà-ü]+)*\b'

def extraire_regex(texte):
    candidats = []
    for mot_cle in MOTS_CLES:
        idx = texte.find(mot_cle)
        if idx != -1:
            extrait = texte[idx:idx+200]
            candidats += re.findall(PATTERN_NOM, extrait)
    return list(set(candidats))
----

==== Liste noire (MOTS_A_IGNORER)

Les faux positifs sont filtrés via une liste noire de ~300 mots :

[source,python]
----
MOTS_A_IGNORER = {
    # Régions et pays
    "Bretagne", "Normandie", "Alsace", "France", "Espagne",
    # Siècles et dates
    "Moyen", "Siècle", "Ancien", "Moderne",
    # Mots grammaticaux
    "Voir", "Forme", "Variante", "Nom", "Prénom",
    # Auteurs et sources
    "Tosti", "Geneanet", "INSEE",
    # ... (~300 entrées au total)
}

== Étape 7 — Normalisation et variantes régionales

=== Partie 1 — Noms composés

==== Problème

Les noms composés existent sous de multiples formes graphiques qui ne sont pas reconnues comme identiques :

* `Le Goff`, `Legoff`, `Le-Goff`
* `Du Bois`, `Dubois`, `Du-Bois`
* `Le Roy`, `Leroy`, `Le-Roy`

==== Solution — Normalisation par agglutination

[source,python]
----
def normaliser_compose(nom):
    return nom.lower() \
              .replace("-", "") \
              .replace(" ", "") \
              .replace("'", "")

# Exemples :
# "le goff"  → "legoff"
# "Le-Goff"  → "legoff"
# "LEGOFF"   → "legoff"
# → toutes identiques après normalisation
----
